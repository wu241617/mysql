mysql事务

  • 当一个业务逻辑需要多个sql完成时，如果其中某条sql语句出错，则希望整个操作都回退
  • 使用事务可以完成退回的功能，保证业务逻辑的正确性

事务的四大特性（简称ACID）

  • 原子性：事务中的全部操作在数据库中是不可分割的，要么全部完成，要么均不执行
  • 一致性：几个并行执行的事务，其执行结果必须与按某一顺序串行执行的结果一致
  • 隔离性：事务的执行不受其他事务的干扰，事务执行的中间结果对其他事务必须是透明的
  • 持久性：对于任意已提交事务，系统必须保证该事务对数据库改变不丢失，即使数据库出现故障

使用要求

  • 要求：表的类型必须是innodb或bdb类型，才可以对此表使用事务

常见使用

  • 查看表的创建语句 
    show create table students
  • 修改表的类型 
    alter table '表名' engine=innodb
  • 事务语句 
                            开启 begin;
                        提交 commit;
                        回滚 rollback;
